Why

You've stopped using AI for half your real work.

Every professional has a folder of documents they'd never paste into ChatGPT. Client contracts. Session notes. Financial docs. Things under NDA. So they use AI for the easy 30% and do the hard 70% by hand.

Purely Personal is for the hard 70%. A free, private AI that runs entirely on your own laptop. Nothing you type ever leaves the machine. No cloud. No subscription. No account.

The point isn't to replace ChatGPT. It's to add the missing half, so you stop choosing between "use AI" and "protect your data."

Install Ollama

Easiest path, works on every Mac:

Download Ollama for Mac Drag to Applications, launch once. No Homebrew needed.

Already have Homebrew? You can use Terminal instead:

brew install ollama
brew services start ollama

Either way, verify it works: curl http://localhost:11434/api/version should return JSON with a version number. If you see "command not found: brew", that means Homebrew isn't installed. Use the download button above instead.

Download the AI model

ollama pull qwen2.5:7b

4.7 GB, 3–10 min on WiFi. Keep the laptop plugged in. Verify with ollama list.

How it works

Three open-source pieces, each doing one thing:

What is Ollama?

The runtime. A small open-source program (~40 MB) that loads AI models into RAM and runs them. It exposes a local HTTP API at localhost:11434. Think of it as a local-AI Docker.

It does five things: pull models, load them into memory, serve an HTTP API, stream tokens as they generate, and use your GPU if you have one (Apple Silicon's Metal, Nvidia's CUDA).

Phones home only once: the first ollama pull. After that, zero outbound connections. Verify: lsof -i -n -P | grep ollama.

What is Qwen 2.5?

The model. 7 billion parameters, trained by Alibaba's Qwen team, released with open weights (free to use commercially). Quantized to 4-bit = 4.7 GB on disk.

Competitive with GPT-3.5 on most tasks, GPT-4-class on code and summarization. Weaker on hard reasoning and very long documents (>32k tokens).

The file is pure numbers. No executable code, no telemetry, no callbacks. If you hex-dump it you'll see matrices of tiny 4-bit integers. That's the whole "brain."

What is Purely Personal?

The app. An Electron desktop wrapper (~95 MB) that talks to Ollama over the local loopback. Adds chat streaming, drag-and-drop, saved prompts, conversation history, 23 templates, folder indexing (RAG), and dark mode.

Built in ~8,000 lines of code. MIT-licensed. Fork it, read it, extend it.

How does "Index a folder" (RAG) work?

Pick a folder → app walks every text file → breaks them into 900-char chunks → embeds each chunk (using nomic-embed-text through Ollama, locally) → stores chunks + vectors in localStorage.

When you ask a question, it embeds your question, finds the 5 most similar chunks via cosine similarity, and hands those to Qwen 2.5 as context. The model answers using only those chunks. Citations included.

All of it happens locally. No OpenAI embeddings API. No Pinecone. No cloud vector DB.

Why does it actually work offline?

Because the only network hop is localhost:11434, which is your own computer talking to itself. It doesn't need a router, a DNS server, or any outbound connection.

Prove it three ways:

  • Easy: Turn off WiFi. Ask a question. It answers.
  • Stronger: Activity Monitor (Mac) or Resource Monitor (Windows), filter by Purely Personal, chat for 10 min. Bytes to external IPs: zero.
  • Paranoid: sudo lsof -i -n -P | grep -E "ollama|Purely" during a chat. Loopback only.

If something breaks

Quick fixes

"Ollama not reachable" in the app

Ollama isn't running. Start it:

Mac: brew services restart ollama

Linux: sudo systemctl restart ollama

Windows: quit Ollama from the system tray, then relaunch from Start menu, or run ollama serve in PowerShell.

"No models installed" in the app

The pull didn't finish. Run it again:

ollama pull qwen2.5:7b

It resumes automatically if it failed partway. Disk-space error? Free at least 10 GB.

Mac: "Purely Personal can't be opened, unidentified developer"

Right-click the app in Applications → Open → Open. One time only.

If right-click doesn't show Open: System Settings → Privacy & Security, scroll down, click Open Anyway next to the Purely Personal line.

Windows: "Windows protected your PC"

Click More info, then Run anyway.

Windows: "fatal: could not create work tree dir · Permission denied"

Two problems at once:

  • You're running PowerShell from C:\WINDOWS\system32. Windows protects that folder from regular users, so git clone can't write anything there.
  • You don't need git clone on Windows. That step is only for Linux. On Windows, you download the .exe installer from the GitHub release.

The fix:

  1. Close the current PowerShell window.
  2. Open a new one (no Administrator needed).
  3. Follow the Windows install tab at the top of this page. Don't clone the repo.

If you want to clone the repo (optional, for source code reading): first run cd $HOME to move out of the system folder, then clone.

My laptop is slow (8 GB RAM)

The first message after launch takes 15–30 seconds while the model loads into memory. Normal. After that, warm responses are instant.

Still unusable? Swap for a smaller model:

ollama pull qwen2.5:3b

Then in the Purely Personal app, click the model dropdown (top-right) and pick qwen2.5:3b. Smaller = faster but less capable.

Corporate laptop with admin restrictions

If brew, sudo, or the installers need admin rights you don't have: use WSL2 on Windows, or ask IT for a one-off exception tonight. Worst case, come to class and watch, we'll sort your install in the first 5 minutes.

FAQ

Common questions

Is this really local, or just marketed as local?

Really local. Three ways to verify: (1) turn off WiFi during the session and the AI keeps working. (2) Activity Monitor shows zero outbound traffic from the app. (3) The whole codebase is open source, read every line.

This isn't a policy claim. It's observable.

Is it as smart as ChatGPT?

For the things it's good at (summarization, rewriting, explaining, quick brainstorming, code review on files under 5k lines), you won't notice the difference in quality for most tasks.

For hard reasoning, current-events, or very long documents (100k+ tokens), ChatGPT and Claude are meaningfully better. The session teaches you the line between the two.

Why free? What's the catch?

No catch. Ollama is open source (MIT). Qwen 2.5 is open-weights (Alibaba released it). Purely Personal is MIT-licensed. Built on open source. Stays free forever. No "pro tier" coming.

What are the limits I should know about?

Honest list:

  • Hard multi-step reasoning: GPT-4/Claude 4 are meaningfully better.
  • Current events: Training cutoff was mid-2024.
  • Very long docs: 32k-token context (about 120 pages). Beyond that, use cloud.
  • PDFs/Word natively: Text files only for now. PDF/DOCX is on the v0.2 roadmap.
  • Images/audio/video: Text-only model. Vision/audio models exist, just not bundled in this app yet.
  • Old hardware: Needs 8+ GB RAM. Below that, use the 3B smaller model.
Can I use this for work / is it legal?

Yes, for almost every company. All three pieces ship with commercial-use licenses. No data leaves your machine, so there's no "sending company data to a third party" problem.

That said, always check your IT policy. Some regulated industries ban any AI tool until their lawyers approve it.

Can I swap to a different model?

Yes. Any model Ollama supports. Once pulled, it shows up in the model dropdown in the top-right of the app.

ollama pull llama3.1:8b
ollama pull mistral:7b
ollama pull phi3:medium
ollama pull gemma2:9b
ollama pull codellama:13b
